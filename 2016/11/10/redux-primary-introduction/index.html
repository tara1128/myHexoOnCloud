Redux初识概要

考虑到复杂应用中JS要管理更多的状态（state），例如UI状态、是否显示某个效果，是否激活某个路由，是否点击了某个Tag、是否有服务器响应等等。由此出现了Redux来做应用的数据和状态管理。初学此物，其设计模式略感绕。记录要点如下，方便日后查阅。

单一数据源：整个应用的state只存在于唯一的store中

改变state的唯一方法是触发action （store.dispatch）

纯函数reducer修改state：接收旧state和action，计算后返回新state

以上是Redux的三大原则。

代码看晕时，请默念这三大原则。

下面是具体一点的内容：

action

  • 一个普通JS对象
  • 用来描述“发生了什么”（没有说怎样更新state）
  • 大概可以长这个样子： { type: ‘INCREMENT’ }
  • 必须有type属性（其值基本为常量）
  • 用store.dispatch({ type: ‘INCREMENT’ })方法把action传给store

reducer

  • 就是一个纯函数：接收旧state和action，返回新state。(previousState, action) => newState
  • 不要在reducer里修改参数
  • 不要在reducer里请求API
  • 不要在reducer里做路由跳转
  • 不要在reducer里调用非纯函数例如Date.now() Math.random()
  • 拆分的多个reducer可以用combineReducers()来合并
