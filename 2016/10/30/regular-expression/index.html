正则表达式的常用要点小结

Regular Expression，RegExp，正则表达式，作为一种常用工具，虽然长得很像外星文，但如果写得好，可以极大的提高代码执行效率。正则表达式的功能就是执行字符串的查找和替换，正确的使用正则，可以避免繁琐的代码循环。

本文记录一些常用的要点，便于工作时查阅。

常见字符

. : 匹配任意字符，换行符除外
\d : 等价于[0-9]，匹配数字
\D : 等价于[^0-9]，匹配非数字
\w : 等价于[A-Za-z0-9_]，别忘了下划线！
\W : 等价于[^A-Za-z0-9]
\s : 匹配空白符，包括空格、制表符、换行符、换页符（\f\n\r\t\v）
\S : 匹配非空白符
\t : 匹配水平制表符tab
\v : 匹配垂直制表符vertical tab
\r : 匹配回车符carriage return
\n : 匹配换行符linefeed
\f : 匹配换页符form-feed
[\b] : 匹配退格键backspace，与\b不同。
\0 : 匹配一个NUL字符
\cX : X为A-Z中的字符，匹配control-X

常见边界

^ : 匹配字符串的开始，/^A/匹配’Anta’，不匹配’antA’
$ : 匹配字符串的结尾，/t$/匹配’eat’，不匹配’tea’
\b : 匹配零宽单词边界（字母与空格之间，即匹配\w与\W之间的空间），/\bno/匹配’at noon’中的’no’
\B : 匹配零宽非单词边界（两字母之间，或两空格之间）/\Bon/匹配’at noon’中的’on’
\b 和 \B，只匹配位置，不匹配字符。用\bcat\b匹配都的字符串长度是3，不是5.

重复匹配

+ : 匹配一个或多个，至少一个（+放在字符或字符集的后面，\w+）
* : 匹配0个或多个，可以是0次，可以多次，表示字符可有可无
? : 匹配0个或1个，最多不超1个。例如合法URL的https中的s，可出现0次或1次
{3} : 匹配重复3次，精确次数，\d{3}
{3, } : 匹配至少3个，\d{3, } 匹配至少3个数字
{2,4} : 匹配最少2次，最多4次。

「贪婪型」匹配和「懒惰型」匹配

贪婪型字符，*，+，{n, } 等字符在匹配时的行为模式是“多多益善”，而不是“适可而止”。它们会尽可能的从一段文本中的开头一直匹配到文本末尾，而不是匹配到第一个匹配为止。

解决的方法，就是让贪婪型变成懒惰型，匹配一个即停止。在字符后面加“?”即可。
* =====> *?
+ =====> +?
{n, } ======> {n, }?

分行匹配模式

例如一段代码中有分散的几行是注释（双斜杠”//“开头的一行），如何用分行模式将其匹配出来？正则中的分行匹配模式（multiline mode）即(?m)记号可以让正则的引擎把行分隔符当作一个字符串的分隔符来对待。这时候，^将不仅匹配正常的字符串开头，还将匹配换行符后的开始位置；同理，$将不仅匹配字符串的结尾，还匹配换行符后面的结束位置。
